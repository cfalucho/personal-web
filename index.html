MISSA Website

  • Live
  • GitHub
  • Goal

    The goal was to re-design, re-develop and profitably reduce the cost for MISSA by moving their website from Squarespace to Github pages.

  • Action

    First, I created several wireframe designs to be presented to MISSA eboard members. Once the specific design was accepted, I quickly developed an HTML prototype for testing and evaluation for user requirements. During development, I utilized Bootstrap framework to create a modern and responsive UI design and successfully deployed the website on Github pages.

  • Constraints

    One of the biggest limitations with moving from a CMS platform (Squarespace) to developing a website from scratch ensuring that it is future proof, making it easier to use and navigate for future developers. Other issues were constant changes during development which would extend deadlines.

  • Results

    Overall, the results have been great and the website feels much more modern and responsive. This is because developing a website from scratch has a lot of great benefits like being able to develop a website exactly how you want it. The new MISSA website has helped many students get information on future club events and general meetings.

CPP Event Mobile App

  • QA Doc
  • GitHub
  • Goal

    The goal was to develop an event register mobile app for Cal Poly Pomona students. Students will log in, have a feed of events happening on campus and be able to RSVP. Once they RSVP to their chosen event, they would see a chronological list of events they RSVP'd to. They can also view the university's clubs and their events.

  • Action

    First, a team of developers including myself first discussed on what technology to use. Our choices were Android or React Native. We chose React Native because it used JavaScript language and it was a technology worth learning. We used an agile approach to deliver product requirements step-by-step.

  • Constraints

    Several issues arose. One was implementing a Tinder-like swipe feature using plugins. When adding additional plugins, the mobile app's performance would slow down. Another major issue we ran into was doing integration testing with the back-end system.

  • Results

    The main front-end functionalities of the app were completed on time. These functionalities include login, swiping, RSVPing to an event, and saving the event to the Feed tab. Overall, the design and development were successful.

Calculator

  • Live
  • GitHub
  • Goal

    Build a calculator web application to learn JavaScript and how to access HTML DOM elements.

  • Action

    First, I created the layout and designed the calculator using HTML/CSS. After I completed the design and created the buttons on the HTML side. I used JavaScript's document method querySelectorAll() to select all of the buttons and then store it into a reference variable. Because querySelectorAll() returns a list of <button> I had to use a for loop to access each button tags and use the addEventListener() method to attach an event handler to the specified element.

  • Constraints

    One issue I ran into was not understanding how querySelectorAll() returns value worked. However, implemented a for loop helped access each button individually. For the =, I had to not include it in the for loop because the button does not display on screen but rather indicate the computation of the equation

  • Results

    Building the project has helped me understand some basic knowledge of how JavaScript can manipulate HTML elements using various DOM methods. I find it fascinating that JavaScript can add a wide range of behavior to a website. The overall result has been great and I continue to work on this web application to further my knowledge in JavaScript.
